ScholarSync ERP System Documentation

ScholarSync is a comprehensive Education Resource Planning (ERP) system designed to streamline academic management processes. It provides an integrated platform for administrators, faculty members, and students to interact with educational resources, manage academic activities, and access institutional information.

This documentation provides an in-depth overview of the ScholarSync ERP system, including its architecture, features, security measures, and implementation details.

Project Overview

ScholarSync was developed as a final year project to address the challenges faced by educational institutions in managing their academic operations. The system aims to replace traditional paper-based processes with a digital solution that enhances efficiency, transparency, and accessibility.

Key Objectives

  • Streamline administrative processes
  • Facilitate better communication between faculty and students
  • Provide easy access to academic resources
  • Enable efficient management of courses, schedules, and assessments
  • Implement role-based access controls for secure data management
  • Support real-time tracking of academic progress

System Architecture

ScholarSync follows a modern web application architecture with a clear separation of concerns. The system is built using the Next.js framework, which provides both client and server-side capabilities.

Frontend Architecture

The frontend of ScholarSync is built with React and follows a component-based architecture. Key aspects include:

  • Component Organization: UI components are structured in the /components directory, organized by function and module.
  • Pages: Each page is defined in the /app directory following Next.js 13+ App Router convention.
  • Context API: React Context is used for global state management, particularly for authentication via AuthContext.
  • Client-Side Routing: Next.js provides client-side routing for smooth navigation between pages.
  • Responsive Design: The UI is fully responsive and adapts to various screen sizes using Tailwind CSS.

Backend Architecture

The backend is implemented using Next.js API routes, which provide serverless functions for handling requests. Key aspects include:

  • API Routes: Located in /app/api directory, providing RESTful endpoints for client interactions.
  • Data Access Layer: Prisma ORM is used for database interactions, providing type-safe queries.
  • Authentication: JWT-based authentication for securing endpoints.
  • File Storage: Integration with Cloudinary for file uploads and storage.
  • Route Protection: Middleware for route protection based on user roles and authentication status.

Authentication & Security

ScholarSync implements comprehensive security measures to protect user data and ensure appropriate access controls.

Authentication Flow

  1. Registration: Users register with email and password
  2. Email Verification: Verification link sent to user's email
  3. Admin Approval: Admin approves new user accounts
  4. Login: JWT-based authentication with secure tokens
  5. Role-Based Access: Different UI and permissions based on user role

Security Measures

  • Password Security: Passwords are hashed before storage
  • JWT Authentication: Secure, expiring tokens for session management
  • HTTPS: All communications encrypted via HTTPS
  • API Protection: Server-side validation and authorization checks for all API endpoints
  • CORS Policies: Properly configured Cross-Origin Resource Sharing
  • Input Validation: Comprehensive validation for all user inputs
  • Rate Limiting: Protection against brute force attacks

Implementation Details

Files are handled using Cloudinary's Node.js SDK. The implementation includes:

// Example of file upload utility
export const uploadToCloudinary = async (
  fileDataUri: string,
  fileName = 'file',
  folder = 'scholarsync'
): Promise => {
  try {
    // Upload to Cloudinary
    const result = await cloudinary.uploader.upload(fileDataUri, {
      resource_type: 'auto',
      folder,
      public_id: `${fileName}_${Date.now()}`,
      transformation: [
        { quality: 'auto' }
      ]
    });
    
    return {
      public_id: result.public_id,
      secure_url: result.secure_url
    };
  } catch (error) {
    console.error('Cloudinary upload failed:', error);
    throw new Error('Failed to upload file');
  }
};

For downloading files, the system ensures that the correct file extension is maintained:

// File download utility that preserves file extensions
const downloadFile = async (url: string, fallbackName: string) => {
  try {
    // Get the filename from the URL
    const urlParts = url.split('/');
    let filename = urlParts[urlParts.length - 1];
    
    // Decode URL encoded characters
    filename = decodeURIComponent(filename);
    
    // If filename doesn't have extension or has query parameters, use fallback
    if (!filename.includes('.') || filename.includes('?')) {
      filename = fallbackName;
    }
    
    // Trigger download
    const link = document.createElement('a');
    link.href = url;
    link.download = filename;
    link.target = "_blank";
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);
  } catch (error) {
    console.error("Error downloading file:", error);
  }
};

Key Workflows

ScholarSync implements several key workflows that facilitate educational management processes.

Student Enrollment Workflow

  1. Admin creates a new course in the system
  2. Admin adds subjects to the course
  3. Admin enrolls students in the course
  4. System assigns the enrolled courses to student accounts
  5. Students can access their enrolled courses

Assignment Submission Workflow

  1. Faculty creates an assessment with a due date
  2. Students receive notification of the new assignment
  3. Students submit their work before the deadline
  4. Faculty reviews submissions and provides grades/feedback
  5. Students receive notification of graded assignments
  6. Students can view their grades and feedback

Attendance Management Workflow

  1. Faculty accesses the attendance module for a specific class session
  2. System displays enrolled students for the class
  3. Faculty marks present/absent status for each student
  4. System stores attendance records
  5. Students can view their attendance statistics
  6. Admin can generate attendance reports

Notice Publication Workflow

  1. Admin or faculty creates a notice with relevant information
  2. Optional: Attaches files to the notice
  3. Sets the target audience (all, department, specific courses)
  4. Publishes the notice
  5. Target users see the notice in their dashboard

The following environment variables need to be configured:

# JWT
JWT_SECRET="your-secret-key"
JWT_EXPIRES_IN="7d"

# Cloudinary
CLOUDINARY_CLOUD_NAME="your-cloud-name"
